# Database Schema Documentation

## Overview

This documentation describes the backend MongoDB schema for the TaskFlow collaborative Kanban board. It is based solely on the current Mongoose models found in `backend/src/models` and documents exact collection structures, validation rules, indexes, and relationships.

## Collections

### Users
- Collection name: `users`
- Purpose: Stores registered application users.
- Fields:
  - `name` (String)
    - required: yes
    - trim: true
    - maxlength: 50
  - `email` (String)
    - required: yes
    - unique: yes
    - lowercase: true
    - trim: true
    - match: email regex `/^\S+@\S+\.\S+$/`
  - `password` (String)
    - required: yes
    - minlength: 6
    - select: false (excluded by default from queries)
  - `createdAt` (Date)
    - auto-generated by timestamps
  - `updatedAt` (Date)
    - auto-generated by timestamps
- Indexes:
  - unique index on `email`
- Relationships:
  - referenced by `Workspace.owner`, `Workspace.members`, `Workspace.memberRoles.user`, `Task.assignee`, `Task.createdBy`, `Task.attachments.uploadedBy`, `Task.comments.author`, `Invitation.invitedBy`, `Notification.user`, `Activity.user`

### Workspaces
- Collection name: `workspaces`
- Purpose: Represents collaboration spaces where tasks and members are organized.
- Fields:
  - `name` (String)
    - required: yes
    - trim: true
    - maxlength: 100
  - `purpose` (String)
    - required: yes
    - trim: true
    - maxlength: 500
  - `inviteCode` (String)
    - unique: yes
    - uppercase: true
    - generated before save if absent
  - `members` (Array of ObjectId refs to User)
    - optional list of workspace member user references
  - `memberRoles` (Array of subdocuments)
    - each item contains:
      - `user` (ObjectId ref to User)
        - required: yes
      - `role` (String)
        - enum: values from `ROLES`
        - default: `member`
  - `owner` (ObjectId ref to User)
    - required: yes
  - `createdAt` (Date)
    - auto-generated by timestamps
  - `updatedAt` (Date)
    - auto-generated by timestamps
- Indexes:
  - unique index on `inviteCode`
- Relationships:
  - `owner` references a User
  - `members` references multiple Users
  - `memberRoles.user` references Users
  - referenced by `Task.workspace`, `Invitation.workspace`, `Activity.workspace`

### Tasks
- Collection name: `tasks`
- Purpose: Stores Kanban tasks and embeds comments and attachments.
- Fields:
  - `title` (String)
    - required: yes
    - trim: true
    - maxlength: 200
  - `description` (String)
    - trim: true
    - maxlength: 2000
    - default: `''`
  - `status` (String)
    - enum: `['todo', 'in_progress', 'review', 'done']`
    - default: `todo`
  - `priority` (String)
    - enum: `['low', 'medium', 'high', 'urgent']`
    - default: `medium`
  - `dueDate` (Date)
    - default: `null`
  - `labels` (Array of String)
    - default: `[]`
    - validation: max length 10 labels
  - `assignee` (ObjectId ref to User)
    - default: `null`
  - `workspace` (ObjectId ref to Workspace)
    - required: yes
  - `createdBy` (ObjectId ref to User)
    - required: yes
  - `attachments` (Array of embedded subdocuments)
    - default: []
    - embedded fields:
      - `_id` (ObjectId)
      - `fileName` (String, required)
      - `originalName` (String, required)
      - `fileType` (String, required)
      - `fileSize` (Number, required)
      - `fileUrl` (String, required)
      - `uploadedBy` (ObjectId ref to User, required)
      - `uploadedAt` (Date, default: Date.now)
  - `comments` (Array of embedded subdocuments)
    - default: []
    - embedded fields:
      - `_id` (ObjectId)
      - `text` (String, required, trim, maxlength 4000)
      - `author` (ObjectId ref to User, required)
      - `mentions` (Array of String, default: [])
      - `edited` (Boolean, default: false)
      - `createdAt` (Date, default: Date.now)
      - `updatedAt` (Date, default: Date.now)
  - `createdAt` (Date)
    - auto-generated by timestamps
  - `updatedAt` (Date)
    - auto-generated by timestamps
- Indexes:
  - compound index on `{ workspace: 1, status: 1 }`
  - compound index on `{ workspace: 1, assignee: 1 }`
- Relationships:
  - `workspace` references Workspace
  - `createdBy` references User
  - `assignee` references User
  - `attachments.uploadedBy` references User
  - `comments.author` references User
  - referenced optionally by `Activity.task`

### Invitations
- Collection name: `invitations`
- Purpose: Stores workspace invitation tokens and email invite details.
- Fields:
  - `workspace` (ObjectId ref to Workspace)
    - required: yes
    - indexed
  - `email` (String)
    - required: yes
    - trim: true
    - lowercase: true
    - match: email regex `/^\S+@\S+\.\S+$/`
  - `inviteCode` (String)
    - required: yes
    - unique: yes
    - trim: true
    - uppercase: true
  - `token` (String)
    - required: yes
    - unique: yes
    - trim: true
  - `invitedBy` (ObjectId ref to User)
    - required: yes
  - `status` (String)
    - enum: `['pending', 'accepted', 'expired']`
    - default: `pending`
    - indexed
  - `expiresAt` (Date)
    - required: yes
    - indexed
  - `createdAt` (Date)
    - auto-generated by timestamps
  - `updatedAt` (Date)
    - auto-generated by timestamps
- Indexes:
  - unique index on `inviteCode`
  - unique index on `token`
  - index on `workspace`
  - index on `status`
  - index on `expiresAt`
- Relationships:
  - `workspace` references Workspace
  - `invitedBy` references User

### Notifications
- Collection name: `notifications`
- Purpose: Stores application notifications for users.
- Fields:
  - `user` (ObjectId ref to User)
    - required: yes
    - indexed
  - `type` (String)
    - required: yes
    - enum: `['workspace_created', 'workspace_joined', 'task_assigned', 'task_completed', 'task_due_soon']`
  - `message` (String)
    - required: yes
    - trim: true
  - `isRead` (Boolean)
    - default: `false`
  - `relatedId` (ObjectId)
    - default: `null`
  - `createdAt` (Date)
    - auto-generated by timestamps
  - `updatedAt` (Date)
    - auto-generated by timestamps
- Indexes:
  - index on `user`
- Relationships:
  - `user` references User

### Activity
- Collection name: `activities`
- Purpose: Stores workspace activity log entries.
- Fields:
  - `user` (ObjectId ref to User)
    - required: yes
  - `workspace` (ObjectId ref to Workspace)
    - required: yes
  - `task` (ObjectId ref to Task)
    - default: null
  - `action` (String)
    - required: yes
    - trim: true
  - `title` (String)
    - required: yes
    - trim: true
  - `description` (String)
    - default: `''`
    - trim: true
  - `timestamp` (Date)
    - default: `Date.now`
  - `createdAt` (Date)
    - auto-generated by timestamps
  - `updatedAt` (Date)
    - auto-generated by timestamps
- Indexes:
  - compound index on `{ workspace: 1, timestamp: -1 }`
- Relationships:
  - `user` references User
  - `workspace` references Workspace
  - `task` references Task

## Relationships Summary

- User -> Workspace (owner)
- Workspace -> User (members)
- Workspace -> User (memberRoles.user)
- Workspace -> Task
- Task -> Workspace
- Task -> User (createdBy)
- Task -> User (assignee)
- Task -> User (attachments.uploadedBy)
- Task -> User (comments.author)
- Workspace -> Invitation
- Invitation -> User (invitedBy)
- User -> Notification
- Workspace -> Activity
- User -> Activity
- Task -> Activity

## Index Summary

- `users.email` unique
- `workspaces.inviteCode` unique
- `tasks` compound indexes: `{ workspace: 1, status: 1 }`, `{ workspace: 1, assignee: 1 }`
- `invitations.inviteCode` unique
- `invitations.token` unique
- `invitations.workspace` indexed
- `invitations.status` indexed
- `invitations.expiresAt` indexed
- `notifications.user` indexed
- `activities.workspace` and `activities.timestamp` compound index

## Design Notes

- `Task` stores `attachments` and `comments` as embedded subdocuments, not separate collections.
- `Workspace.memberRoles` stores role metadata per user inside the workspace document.
- `Invitation` uses unique `token` and `inviteCode` values to guarantee one-time invitation handling.
- `User.password` is excluded from default query results with `select: false`.

## Assumptions

- Collection names are inferred from Mongoose model names in lowercase plural form.
- Embedded `attachments` and `comments` are treated as task-owned subdocuments and not as separate collections.
- `relatedId` on `Notification` is a generic reference; no explicit ref type is enforced in schema.
